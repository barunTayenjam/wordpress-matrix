# Quick Task 260508-001: Fix site logs command

## Changes Made

### matrix (show_logs function)
- Replaced all `logs -f` (follow/tail mode) with `logs --tail=100` (show last 100 lines and exit)
- Affected: db, nginx, redis, phpmyadmin, and per-site log commands

### frontend/app.js
- Added `'logs'` to the `isLongRunning` command list so the frontend grants 300s timeout instead of 30s

## Root Cause
`show_logs()` used `docker-compose logs -f` which never terminates. When the frontend called `POST /api/sites/logs`, the spawned `./matrix logs su21` process ran until the 30-second timeout killed it, causing `ERR_CONNECTION_REFUSED` or silent failures.

## Verification
- `./matrix logs su21` exits cleanly within seconds (not hangs forever)
- `curl -X POST http://localhost:8500/api/sites/logs -H "Content-Type: application/json" -d '{"siteName":"su21"}'` returns `{"success":true,"output":"..."}` within 5 seconds
